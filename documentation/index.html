I. Overview

Munda forms  is a set of form elements build with the help of various frameworks and libraries, that gives you the necessary tools to create any type of web form.

I.I Tools used
  • Node.js
  • Grunt
  • Bower
  • Composer
  • IcoMoon

For the best workflow i recommand that you install Node.js, Bower and Composer.

I.I.I Node.js

Once you have Node.js installed you can run the npm install command which will install all the node modules (like grunt, grunt-contrib-less, grunt-contrib-watch etc), than you will have access to commands likegrunt serve which will launch a server, watch for changes on your files and automatically reload the page upon save. To configure the grunt tasks you can edit the  file.

I.I.II Bower

Bower will allow you to manage frameworks, libraries, assets, utilities etc. Run bower install to install all the dependencies confingured in the bower.json file.

I.I.III Composer

Composer will allow you to manage your PHP dependencies. Runcomposer update in the app/php folder to install the PHP dependencies configured in theapp/php/composer.json file.

I.II.IV IcoMoon

The icomoon selection that i've included with this package has over 800 icons (you can see all the available icons by opening the app/icomoon/demo.html file). I do not recommand that you use this in production, it's best to create your own selection on the IcoMoon website and use those files.

II. CSS

The CSS for this form framework was build using LESS. All the LESS files are located inapp/less folder. The compiled CSS files are located in app/styles(form-framework.cssand form-framework.min.css).

app/less folder content:

  • main.less file - this is where all the other files are imported, here you can comment out the parts of the framework you don't need.
  • jqueryui folder - this is where you can find all the files that style the jQuerUI components.
  • mixins folder - this is where you can find all the LESS mixins used throughout the code.
  • variables folder - this is where you can find all the variables used throughout the code.

app/styles folder content:

  • form-framework.css file - the css for the form framework.
  • form-framework.min.css file - the minified version of form-framework.css.
  • icomoon.css file - this is the css for the font icons. You don't need this file if you don't want to use icons. Also, here you can change the path to your font files, if you have a different site structure than the one i used in the demos.
  • jqueryui-theme.css file - this is the css for jQueryUI. Again, if you don't need any of the jQueryUI's functionality don't include this file.
  • jqueryui-theme.min.css file - the minified version of jqueryui-theme.css.
  • main.css file - this is part of the css for presenting the framework. You don't need this file.
  • main.min.css file - the minified version of main.css.
  • vendor/animate.css file - this the css for CSS animations. Don't include this file if you don't want CSS animations.
  • vendor/grid.css file - this is the css for the grid system, if you already have twitter bootstrap (version 3+) in your site don't include this file.
  • vendor/normalize.css file - makes browsers render all elements more consistently and in line with modern standards. If you already have it in your site, or have twitter bootsrap, or any other framework that comes with this file, or maybe you just don't want to include it, don't do it.
  • vendor/raty-custom.css file - css for the jQuery Raty plugin. If you don't want the rating functionality don't include this file.
  • vendor/shCore.css and vendor/shThemeDefault.css files - this are for the sysntax highlighting plugin. You don't need this files.
  • vendor/spectrum.css and vendor/spectrum-custom.css files - this are the css files for the colorpicker. Don't include them if you don't need colorpicker functionality.

app/styles/vendor/skins folder contains folders for each color variation.

II.IV Form controls

Individual form controls automatically receive some global styling. All textual elements with .form-control are set to width: 100%; by default. Wrap labels and controls in .form-group for optimum spacing.

Every .form-control should have a label. To hide a label and only show it to screen readers you must add the sr-only class.

You can also add the .label class to other elements to give those elements the same styles as a <label> tag.

II.XIX Tables

For basic styling, light padding and only horizontal dividers, add the base class .table to any <table>.
Striped rows

Use .table-striped to add zebra-striping to any table row within the <tbody>.
Bordered table

Add .table-bordered for borders on all sides of the table and cells.
Hover rows

Add .table-hover to enable a hover state on table rows within a <tbody>.
Condensed table

Add .table-condensed to make tables more compact by cutting cell padding in half.
Contextual classes

Use contextual classes to color table rows or individual cells.

  • .active - applies the hover color to a particular row or cell
  • .success - indicates a successful or positive action
  • .info - indicates a neutral informative change or action
  • .warning - indicates a warning that might need attention
  • .danger - indicates a dangerous or potentially negative action

  • html5shiv.js - enables use of HTML5 sectioning elements in legacy Internet Explorer and provides basic HTML5 styling for Internet Explorer 6-9, Safari 4.x (and iPhone 3.x), and Firefox 3.x.
  • respond.min.js - a fast & lightweight polyfill for min/max-width CSS3 Media Queries (for IE 6-8, and more).
  • mootools-core-full-compat.min.js - is a dependency for selectivizr-min.js.
  • selectivizr-min.js - JavaScript utility that emulates CSS3 pseudo-classes and attribute selectors in Internet Explorer 6-8.

The Modernizr library is also included, this will help you detect HTML5 and CSS3 features in the users browser.

In the app/php/ folder you will find 3 fully functional forms( contact.php, request-quote.php and register.php), use these files (and the other included files) as the starting point for your forms.

For the forms to function properly, you need to modify the app/php/creds.php file and link it in (see line 7 in register.php, or line 8 in contact.php and request-quote.php).

V. Quick start

  1. Link the CSS files into your project, i've explained here what each of the file does. Based on that you can decide if you need to include the file into you project.
  2. Link the javascript files, you can see here what each of the library does and you can decide if you need it in your project.
  3. Each of the HTML demos (located in the app/examples folder) has it's own specific javascript file, where all the javascript for that specific form is located. E.g. review.html has a review.js file. Use this HTML files as starting points for you forms.
  4. The fully functional AJAX forms (contact, request-quote, register) are located in the app/php folder. In order to make them work you need to run composer update inside the app/php folder. This command will install all the necessary php dependencies. Next you need to edit the creds.php file with your credentials, then change the line (from contact.php, request-quote.php, register.php, email-checker.php, username-checker.php) that says require_once '../../../../CREDS/index.php'; to require_once 'creds.php';. For the register.php you also need to create a users table in your database with the following columns: id, username, first_name, last_name, gender, email_address, password, subscription, registration_date. I warn you however that this is not a complete login and registration system, this is mostly to demonstrate database interaction.
  5. Read the documentation for more information.
